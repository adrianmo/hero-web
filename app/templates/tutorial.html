Tutorial

VxRack Neutrino nodes is a turnkey multi-service cloud appliance that provides an open source platform for cloud native applications. It delivers web-scale, self-scaling, self-resilient application infrastructure for a secure, elastic private cloud.

In this tutorial, you will:

  • Get to know the Neutrino UI
  • Access OpenStack and perform a simple operation
  • Explore some of the reports that come out of the box, including Performance, Utilization, Inventory, Alerts and Logs

Step 1: Getting to know the Neutrino UI

Once you receive the registration email with your credentials and the login URL, go ahead and log into Neutrino.

Explore the Neutrino dashboard

The Dashboard displays the overall VxRack Neutrino system status. Each tile displays a certain aspect of the VxRack Neutrino system. This view provides a quick read-only overview of the health, cloud compute, and storage resources in the VxRack Neutrino system.

The Dashboard is divided into six tiles:

  1. Health - provides health status information on the VxRack Neutrino infrastructure (nodes, disks, networks, storage) and software services (Platform and Cloud Compute).
  2. Node Allocation - a donut chart displays the used and unallocated (unused) nodes in the VxRack Neutrino system using colored segments.
  3. System - displays information about the installed version, the aggregate memory of all the nodes and the aggregate block storage capacity of all the nodes in the VxRack Neutrino system.
  4. Alerts - lists notifications for VxRack Neutrino system components that may require the Cloud Administrator's attention. Each alert listed contains a brief message description, timestamp, and severity.
  5. Cloud Compute Service - provides information on the OpenStack environment: Instances (Virtual Machines), vCPUs, Memory and Storage
  6. Storage - provides information on block storage capacity use for cloud compute purposes via a donut chart and IOPs/bandwidth information for the block storage system via a dynamic graph

Open Horizon Dashboard

Navigate to Services->Cloud Compute and click on Openstack Dashboard button. This will open a new tab with the Horizon dashboard. You will be already logged in with your user.

Step 2: Openstack operations

Now that you are in the Horizon UI, let's create an instance (VM) in Openstack. Under Compute, click on Instances, and then click on Launch Instance

You are now presented with the Launch Instance wizard. Enter a name for your instance, for example hero-vm and click Next.

In the next screen you select the image that will be used to boot the VM. Click on the plus (+) sign next to hero-ubuntu and click Next.

Next screen is where you specify the flavor for the VM, in other words, the resources that the VM will be created with. Select m1.tiny and click Next.

In the next screen of the wizard you select what networks will the VM be connected to. Select default_network and click Next four times until you get to the Configuration step.

Step 3: Explore the reports

Now that your VM is ready, you will go back to the Neutrino UI and explore some of the reports about Performance, Capacity and Chargeback in your Openstack cloud.

Go back to the tab were you had the Neutrino UI. Then navigate to Reports->Performance

Performance Reports

Performance Reports are organised in three categories: Infrastructure, Cloud Compute and Accounts. Infrastructure is subdivided in Storage, Network and Platform. Take some time to explore these reports. Here we show the Infrastructure->Network view that provides performance details for all network switches and ports in the platform

Now move to Cloud Compute. This set of reports will show you performance information about your Openstack environment. It is subdivided in Nodes (Hypervisors), Instances, CPU Utilization and Memory Utilization. Feel free to navigate around.

Capacity Reports

You will now explore the capacity related reports. Navigate to Reports->Capacity.

This report category is subdivided in three different categories: Infrastructure, Cloud Compute and Accounts. It provides information about capacity consumption and forecast of all components in the platform. Feel free to navigate around. Here we show an example screenshot of the "Cloud Compute->Compute Nodes current report". This page provides several tables with the current allocation of all nodes in the system.

Chargeback Reports

Go back to the tab were you had the Neutrino UI. Then navigate to Reports->Chargeback

You will take a look to the "Default Account" report. Click in Reports->ChargeBack->Default.

This report will show you information about your associated "Default Account" costs. It includes timing reports per Resource type: CPU, Memory, Network and Storage, Total costs per project or total costs per time period.

You can navigate on these reports by clicking on different elements exhibited on this "Account Default" report. This page, "as example", provides detaild information about "mosaic Project" with the cost metrics for this entire project

Inventory Reports

Go back to the main Neutrino UI page cliking on Reports->Inventory tree option on the left of the screen

Now move to Cloud Compute tab. This set of reports will show you inventory information about your Openstack environment. It is subdivided in Instances (VMs), Networks and Image Sumamary. Feel free to navigate around.

Do you believe this is the end? This is just a small part of what you can discover around VxRack Neutrino. Feel free to navigate any other section of the UI, like Logs, Health reports, etc.
